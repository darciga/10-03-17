Angular Validación de formularios

Este módulo nos ayuda específicamente a mostrar mensajes de error desde la validación de formularios, y nos apoyaremos con las propiedades que nos proporciona Angular en los formularios y nos que nos ayudan a validarlos.

Propiedad Clases Descripción
$valid ng-valid Boolean Indica si un elemento es actualmente válido según las reglas que ha colocado.
$invalid ng-invalid Boolean TIndica si un elemento es actualmente inválido según las reglas que ha colocado.
$pristine ng-pristine Boolean True si el formulario/input no ha sido utilizado aún.
$dirty ng-dirty Boolean True si el formulario/input has ha sido usado.
$touched ng-touched Boolean True si el inputo has ha sido "bluereado".

ACCESO A PROPIEDADES DE FORMULARIO ANGULAR

  • Para acceder a las propiedades del formulario: <form name>.<angular property>
  • Para acceder a las propiedades del input: <form name>.<input name>.<angular property>

Validación de reglas

Angular proporciona muchas reglas de validación que podemos usar además de ng-minlength y ng-maxlength.

Estos son los parámetros disponibles para una entrada Angular para crear reglas de validación.

<input
  ng-model="string"
  [name="string"]
  [required="string"]
  [ng-required="boolean"]
  [ng-minlength="number"]
  [ng-maxlength="number"]
  [ng-pattern="string"]
  [ng-change="string"]
  [ng-trim="boolean"]>
...
</input>
